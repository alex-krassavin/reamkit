## W1 — Гиперссылки: модель → PDF `/Link` + HTML `<a>` ✅ (выполнено)

**Спека:** ECMA-376 §17.16.22 (`w:hyperlink`: `r:id` → external rel c
`TargetMode="External"`, `w:anchor` → внутренняя закладка); ISO 32000
§12.5.6.5 (Link annotation) + §12.6.4.7 (URI action).

**Сейчас:** `w:hyperlink` в `RUN_CONTAINER_TAGS` (document-parser.ts:48) —
текст сохраняется, семантика ссылки теряется молча. `/Annots` в эмиттере
существует только для подписного виджета на странице 1.

**Дизайн:**

1. **Модель:** `Run.href?: string` — абсолютный URL, унаследованный всеми
   ранами контейнера. Внутренние якоря (`w:anchor`) в v1 остаются текстом
   (закладки/bookmarkStart не парсятся — отдельная фича W5).
2. **Парсер:** ParseContext получает `resolveHyperlink(rId) → url | undefined`
   (из rels части, только `TargetMode="External"`); разворачивание контейнера
   проставляет `href` на дочерние раны.
3. **Прокидка в layout:** RunPlan → `TextToken.href?` — добавление
   ОПЦИОНАЛЬНОГО поля в замороженную схему PageDoc: не ломает ни одного
   потребителя (svg игнорирует, эмиттер читает) — допустимое расширение
   @experimental-схемы в миноре.
4. **PDF-эмиттер:** при проходе токенов строки собирать интервалы по href
   (x-старт/x-конец подряд идущих токенов одной ссылки; rect по
   baseline±ascent/descent строки, в y-up фрейме после флипа). Обобщить
   `/Annots`: `pageAnnots: Map<pageIndex, PdfRef[]>`, подписной виджет
   вливается туда же. Аннотация: `/Subtype /Link`, `/Rect`,
   `/Border [0 0 0]`, `/A << /S /URI /URI (encoded) >>`.
5. **Tagged PDF:** в tagged-режиме Link-аннотация обязана сидеть в структуре
   (Link StructElem + OBJR, §14.8.4.4.2; Matterhorn 28-011) — добавить
   Link-узел вокруг MCID токенов ссылки + OBJR на аннотацию; проверить
   veraPDF 1a/2a/3a.
6. **HTML-writer:** `runHtml` оборачивает span в `<a href="…">`.
7. **БЕЗОПАСНОСТЬ (обязательно):** allowlist схем `http:`/`https:`/`mailto:`
   в ОБОИХ writer'ах. `javascript:`, `data:`, `file:`, vbscript и
   протокол-relative — ссылка не эмитится (текст остаётся), loss
   `degraded/hyperlink`. Документ — недоверенный вход; `<a href>` и PDF /URI —
   исполняемые поверхности.
8. **Фикстуры:** build-docx расширить опцией `hyperlinkRels`
   (rId → URL, TargetMode External) по образцу images.

**Гейт:** нет ссылок → нет `href` → нет аннотаций → байт-в-байт. Тесты: PDF
(`/Subtype /Link`, `/URI`, rect на нужной строке, мультистрочная ссылка ⇒ ≥2
rect'а), HTML (`<a href`), санитайзер (`javascript:` не эмитится + loss),
tagged (veraPDF зелёный). Усилие: **среднее**.

## W4 — Table styles (`w:tblStyle` + conditional formatting) ✅ (выполнено)

**Спека:** §17.7.6 (table styles в styles.xml: tblPr/tcPr/rPr/pPr на стиле),
§17.4.62 `w:tblLook` + условные форматы (firstRow/lastRow/firstCol/lastCol/
band1Horz/band2Horz…).

**Сейчас:** styles-parser читает paragraph/character-стили; table-стили
пропускаются. Реальный договор (#137) показал недостающие шапки/банды.

**Дизайн — по лекалу S6-3 (резолв в reader):** table-style каскад резолвится
в docx-reader'е, cell/row/table properties в FlowDoc становятся финальными —
layout, PDF и HTML writer'ы НЕ меняются вообще. Порядок слоёв (§17.7.6.6):
docDefaults → table style (basedOn-цепочка) → условные форматы по позиции
ячейки (tblLook-маски) → прямые tblPr/trPr/tcPr. Распространяются: borders,
shading, cell margins, и rPr/pPr слоя стиля в каскад ранов ячейки (до
resolveBodyStyles).

**Гейт:** таблицы без `tblStyle` — байт-в-байт; с ним — новые тесты
(banded-стиль: чётные/нечётные строки, firstRow bold+shading; tblLook
отключает банды). byteRisk низкий. Усилие: **среднее** (чистый parse-слой).

## W2 — Поля PAGE / NUMPAGES в колонтитулах ✅ (выполнено)

**Спека:** §17.16.5.35 (PAGE), §17.16.5.33 (NUMPAGES); fldSimple (§17.16.19)
и complex fields `w:fldChar begin / w:instrText / separate / end`
(§17.16.18).

**Сейчас:** `w:fldSimple` — run-контейнер: рендерится ЗАПЕЧЁННЫЙ кэш
результата из исходника (для PAGE это часто «1» на всех страницах). Это
видимая часть HF-drift'а на корпусе.

**Дизайн:**

1. **Парсер:** распознать instr `PAGE`/`NUMPAGES` в fldSimple и в
   complex-полях (конечный автомат begin→instrText→separate→(cached
   result)→end; для остальных инструкций — текущее поведение, кэш). Модель:
   `Run.field?: 'PAGE' | 'NUMPAGES'` (текст рана = плейсхолдер).
2. **Рендер:** band'ы колонтитулов пре-рендерятся один раз на секцию — поле
   делает band динамическим. Вариант: band БЕЗ полей идёт старым путём
   (байт-в-байт); band С полем помечается dynamic и пере-layout'ится НА
   СТРАНИЦУ после пагинации (страницы уже собраны ⇒ и номер, и total
   известны; подстановка текста меняет ширины ⇒ нужен честный re-layout
   band'а, не подмена глифов). Команды dynamic-band'а вставляются в
   `page.commands` постфактум, теми же позициями, что у статического.
3. **HTML:** в потоке страниц нет; поля в body (редкость) остаются кэшем как
   сейчас.

**Гейт:** band'ы без полей — байт-в-байт; **обязательный корпус-прогон**
(HF-доки — чувствительная зона: ждём улучшение TextSim на header/footer
сьюте, ноль регрессий). byteRisk: **высокий** в HF-зоне. Усилие: среднее+.

## W3 — Сноски (footnotes / endnotes) ✅ (выполнено)

**Спека:** §17.11 (`footnotes.xml`/`endnotes.xml`, `w:footnoteReference`,
сепаратор id=0/continuation id=1), tagged: Note/FENote (PDF/UA).

**Сейчас:** не парсится ничего; текст сносок теряется целиком.

**Дизайн (v1, осознанные упрощения):**

1. **Модель:** FlowDoc получает `footnotes?: Map<id, BodyElement[]>` (+
   endnotes); `Run.footnoteRef?: id`. Reader загружает части, прогоняет их
   через те же трансформы (numbering + resolveBodyStyles).
2. **Layout:** ссылка рендерится superscript-номером (verticalAlign уже
   есть). Размещение greedy: когда строка со ссылкой ложится на страницу,
   контентная высота страницы уменьшается на высоту блока сноски (сноска
   layout'ится в ширину контента заранее); разделительная линия ~2".
   Перенос длинной сноски на следующую страницу — v2 (в v1 сноска атомарна,
   как shape). Endnotes — блок в конце документа (дёшево).
3. **Tagged:** Note StructElem со ссылочным P; v1 без FENote-роли (PDF/UA —
   отдельный этап).
4. **HTML:** идеально ложится в поток — `<sup><a href="#fn1">1</a></sup>` +
   `<ol class="footnotes">` в конце; обратные ссылки `#fnref1`. Внутренние
   `#`-якоря безопасны (не URL-схема).

**Гейт:** документы без сносок — байт-в-байт; корпус-прогон (в POI есть
footnote-доки); veraPDF tagged. Усилие: **большое** (новый layout-механизм —
самое близкое к A9 PageAssembler: возможно, начать с его выноса).
